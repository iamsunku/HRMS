## Redis cache (optional but recommended)

This project supports optional Redis caching for improved API performance. To enable caching:

1. Provision a Redis instance (locally with `redis-server`, or a managed provider).
2. Set the `REDIS_URL` environment variable (example: `redis://localhost:6379`).
3. (Optional) set `SEED_SECRET` in `.env` if you plan to use seed endpoints.

Quick check: run the bundled script to verify connectivity and a basic set/get:

```bash
export REDIS_URL=redis://localhost:6379
npm run check-redis
```

If Redis isn't configured, the app will fall back to an in-memory cache for local development.
